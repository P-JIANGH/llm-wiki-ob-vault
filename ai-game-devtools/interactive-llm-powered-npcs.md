# Interactive LLM Powered NPCs

An open-source project enabling **realistic dynamic dialogue with LLM-powered NPCs in any game** (Cyberpunk 2077, Assassin's Creed, GTA 5, etc.). Works by replacing facial pixels generated by the game with AI-generated facial animations — no game source code modification required.

## Overview

The project transforms static, repetitive NPC conversations into dynamic, context-aware interactions. Players use their microphone to speak, listen to NPC voices, and witness lifelike facial animations — all generated in real-time.

**Target games**: Previously released open-world titles unlikely to receive official LLM dialogue features. The project currently includes a Cyberpunk 2077 integration as reference.

## Technical Architecture

### Pipeline

1. **Microphone** → SpeechRecognition (speech-to-text)
2. **DeepFace** facial recognition → identify which NPC is on screen
3. **ChromaDB** vector lookup: character knowledge DB + world/public knowledge DB
4. **Cohere LLM** (or GPT-4 via code modification) + **LangChain** → generates context-aware response using character personality, pre-conversation dialogue style, and retrieved context
5. **Edge-TTS** (or custom voice.py) → text-to-speech audio
6. **SadTalker** → face image + audio → lip-synced facial animation video
7. **Game overlay** → replace NPC face pixels with generated animation

### Key Modules

| File | Purpose |
|------|---------|
| `main.ipynb` | Main interaction loop — player_name, game_name, interact_key params |
| `create_public_vectordb.ipynb` | Build world knowledge vector DB per game |
| `create_character_vectordb.ipynb` | Build per-character knowledge vector DB |
| `create_face_recognition_representation.ipynb` | Train face embedding for named characters |
| `SadTalker/` | Facial animation submodule (separate model download required) |
| `functions/` | Personality creation, video mode, audio mode processing |
| `Cyberpunk_2077/` | Example integration with named characters (Jackie Welles, etc.) |

### Dependencies

- **LLM/Agent**: `cohere`, `langchain` — LLM agent framework with ChromaDB vector stores
- **Vision**: `deepface` — facial recognition, detection, gender/age/emotion analysis
- **Audio**: `edge-tts` (TTS), `SpeechRecognition` (STT), `pydub`, `pyaudio`
- **Animation**: **SadTalker** — audio-driven facial animation (submodule)
- **Storage**: `chromadb` — local vector database for character + world memory

### Game Integration Pattern

1. Create `GameName/` folder with `world.txt` (world description) + `public_info.txt` (public knowledge)
2. `characters/default/` — background NPC template with `pre_conversation.json` (common dialogue)
3. `characters/CharacterName/` — named NPC with: `images/`, `bio.txt`, `character_knowledge.txt`, `pre_conversation.json`, `conversation.json`, `voice/voice.py`
4. Run notebooks to build vector DBs
5. Launch `main.ipynb`, aim game camera at NPC, hold interact key, speak

### Emotion Recognition

Webcam captures player's facial expressions via DeepFace, allowing NPCs to adapt responses based on player emotion — adding a bidirectional emotional layer to the interaction.

## Key Features

- **No game modding required** — pixel replacement overlay works with any game
- **Named NPC support** — character-specific knowledge, personality, voice, and conversation history
- **Background NPC generation** — LLM generates unique name + personality for unnamed NPCs
- **Non-visual mode** — audio-only for horseback/combat scenarios (say NPC name first)
- **Pre-conversation style injection** — character's iconic lines shape LLM dialogue style
