# Game Design Document (GDD)

## Overview
游戏设计文档（GDD）是游戏的总体规划，用于传达愿景、描述内容、制定实施计划。对于螨光游戏工作室这样的小团队，GDD 是控制范围、保持焦点的关键工具。

## 核心功能
1. **传达愿景** — 让所有人理解游戏是什么
2. **保持团队一致** — 新成员快速上手
3. **组织思路** — 整理想法，跟踪变化

## One Pager（单页版）

简化的游戏概念快照，类似海报 + 快速摘要：
- 标题和标语
- 核心概念一句话说明
- 类型、目标用户、平台
- 2-3 个参考游戏

**适用**：脑暴会议、快速沟通、题材验证、策划整理。

## 完整 GDD 结构

### 1. 基础信息 / Elevator Pitch
- **标题与标语**
- **一句话 pitch**：这是什么游戏，为什么它很酷
- **类型**: 模拟、策略、RPG、平台跳跃...
- **目标用户**: 年龄段、偏好
- **平台**: PC / 移动 / Web / 主机
- **灵感来源**: 游戏、电影、书籍

### 2. 游戏玩法
- **核心循环** (Core Loop)：玩家反复做什么
  - 例: 探索 → 获取资源 → 制造 → 使用 → 探索
- **关键操作**：主要动作和控制方案
- **规则与系统**: 计分、进度、经济、AI、物理
- **关卡设计**: 世界结构、挑战、奖励

### 3. 故事与世界
- **剧情概览**
- **角色**: 主角、盟友、敌人、NPC
- **世界设定**：时代、场景、氛围
- **主题**: 友谊、生存、贪婪...

### 4. 视觉与感觉
- **视觉风格**: 写实、卡通、像素艺术、手绘
- **角色设计**
- **场景设计**
- **UI/UX**: 菜单、血条、地图、交互设计

### 5. 音频
- **音乐**: 风格、情绪
- **音效**: 脚步声、爆炸、技能...
- **音效**: 配音需求

### 6. 技术（可选）
- **引擎与工具**: Phaser 3 + Vue 3 + TypeScript + Vite
- **技术限制**
- **变现方式**: 一次性购买 / 内购 / 广告

### 7. 开发计划
- **时间线**: 原型 → Alpha → Beta → 发布
- **团队角色**

## 实用技巧
- **保持简洁** — 越短越好，只写必要的
- **用玩家视角写** — 描述“玩游戏时感受到什么”
- **切割无关内容** — 谜题游戏不需要 20 页角色背景
- **获取反馈** — 让不熟悉想法的人阅读，问题即黄金
- **加入图表** — 流程图、草图、UI 原型
- **它是活文档** — 随着开发演进

## 针对螨光工作室的建议

作为小团队（甚至独立开发者），GDD 的核心目的是**控制范围**。

- 先写 One Pager 验证题材
- 确定 Core Loop 后再扩展其他系统
- 把 GDD 当作**“不做什么”的清单**，而不仅是**“做什么”的清单**
- 每个项目存一个 GDD 文件，用 Git 跟踪版本变化
