## 核心定位

GameGen-O 是**首个专为生成开放世界游戏视频而设计的 Transformer 模型**，被业内称为"游戏工作室的 ChatGPT 时刻"。

主要功能：

- **游戏角色生成**：一键生成西部牛仔、太空人、魔法师、警卫等角色
- **游戏环境生成**：可替代实地取景，降低开发成本
- **动作生成**：支持各种人称视角的动作视频
- **事件生成**：海啸、龙卷风、火灾等游戏事件
- **交互控制**：通过文本、操作信号和视频提示控制游戏内容
- **开放域生成**：不限风格、环境、场景

## 技术架构

### 两阶段训练

#### 阶段一：基础预训练

- 使用 **2+1D VAE**（变分自编码器，如 Magvit-v2）压缩视频片段
- 混合帧速率和分辨率训练策略
- 架构参考 Latte 和 OpenSora V1.2 框架
- 掩码注意力机制实现：文本到视频生成 + 视频续接

#### 阶段二：指令调整（InstructNet）

- 固定预训练模型
- 微调 InstructNet 分支
- 支持多模态输入：结构化文本、操作信号、视频提示
- 建立当前片段与未来片段的映射关系，实现**多模态交互控制**

### 技术要点

- **VAE 解码器**：针对游戏领域进行特定调整
- **InstructNet**：允许用户基于当前片段不断生成和控制下一个片段
- **解耦标签（decoupled labeling）**：描述片段内容状态变化，增强交互性

## 数据集：OGameData

腾讯为训练 GameGen-O 构建了专用数据集管道：

```
原始视频收集 → 人类专家筛选 → 场景检测切割 → 质量排序过滤 → GPT-4o 注释
```

**OGameData 数据集规模：**

| 指标 | 数量 |
|------|------|
| 原始视频 | 32,000 个（来自数百款开放世界游戏） |
| 筛选后可用 | ~15,000 个 |
| 游戏类型 | 角色扮演、第一人称射击、赛车、动作益智等 |
| GPT-4o 注释量 | 4,000+ 小时高质量片段 |
| 分辨率 | 720p 至 4K |

数据来源：100+ 款开放世界游戏，时长从几分钟到几小时不等。

## 交互控制方式

GameGen-O 支持三种多模态交互控制：

1. **文本提示**（Text Prompt）：通过文字描述生成画面
2. **操作信号**（Action Signal）：如 WASD 键盘输入控制角色移动
3. **视频提示**（Video Hint）：类似 ControlNet，通过参考视频引导生成
