## Overview

An open-source project enabling realistic dynamic dialogue with LLM-powered NPCs in any game (Cyberpunk 2077, Assassin's Creed, GTA 5, etc.). Works by replacing facial pixels generated by the game with AI-generated facial animations — no game source code modification required.

## Key Features

- **Voice Input**: Microphone → speech-to-text transcription
- **Facial Recognition**: DeepFace identifies which NPC the player is interacting with
- **LLM Response**: Cohere LLM (or GPT-4) generates context-aware dialogue using LangChain + ChromaDB vector stores
- **Facial Animation**: SadTalker generates lip-synced facial animation video from NPC's extracted face image + TTS audio
- **Emotion Recognition**: Webcam captures player facial expressions → NPCs adjust responses
- **Non-Visual Interaction**: Audio-only mode for horseback/combat scenarios (say NPC name first)
- **Pre-Conversation Files**: Character-specific dialogue style via iconic lines in `pre_conversation.json`
- **Memory**: ChromaDB vector stores for character knowledge + world knowledge

## Architecture

### Core Pipeline

1. Microphone input → SpeechRecognition (text)
2. DeepFace facial recognition → identify NPC
3. ChromaDB lookup: character vector DB + world/public vector DB
4. LLM (Cohere/GPT-4) generates response using context + pre-conversation lines
5. Edge-TTS text-to-speech → audio file
6. SadTalker: face image + audio → facial animation video
7. Game overlay: replace NPC face pixels with generated animation

### Key Files

- `main.ipynb` — main interaction loop (player_name, game_name, interact_key params)
- `create_public_vectordb.ipynb` — build world knowledge vector DB per game
- `create_character_vectordb.ipynb` — build per-character knowledge vector DB
- `create_face_recognition_representation.ipynb` — train face representation for named characters
- `SadTalker/` — embedded submodule for facial animation (requires separate model download)
- `functions/` — personality creation, video mode, audio mode processing
- `Cyberpunk_2077/` — example game integration with named characters (Jackie Welles, etc.)

### Dependencies

- `cohere`, `langchain`, `chromadb` — LLM agent and vector store
- `deepface` — facial recognition, detection, emotion analysis
- `opencv-python` — image/video processing
- `edge-tts` — text-to-speech (default voice engine)
- `SpeechRecognition` — speech-to-text
- `pydub`, `pyaudio` — audio handling
- SadTalker (submodule) — facial animation

## Game Integration Pattern

1. Create game folder: `GameName/`
2. `world.txt` — game world description; `public_info.txt` — public knowledge
3. `characters/default/` — background NPC template (pre_conversation.json)
4. `characters/CharacterName/` — named character with: images/, bio.txt, character_knowledge.txt, pre_conversation.json, conversation.json, voice/voice.py
5. Build vector DBs via notebooks
6. Run main.ipynb with game name
